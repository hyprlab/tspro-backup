{{ icon('shield') }} End-to-end encryption

Each site gets its own keypair. TS Pro encrypts every backup to the site's public key before upload; only the matching private key — shown once at site creation and never stored here — can decrypt it. This server holds only ciphertext, so it can't read your backups even if fully compromised. When required, the API rejects any upload that isn't already encrypted, so plaintext can never land here.

{{ icon('lock') }} Server-side encryption at rest

Defense-in-depth for the storage volume: wraps each stored blob with a key this server holds (TSPB_REST_PASSPHRASE / rest.key). This is not end-to-end — the server can remove this layer. Leave off if you rely on E2EE above.

{{ icon('clock') }} Default retention

Default grandfather-father-son policy per site & scope. Keep the newest backup of each recent day / week / month / year. 0 disables a tier.
